## 3. 피드백 1차

**User:**
> 1. Github로 관리할거라 즉각 수정/삭제 반영이 안될 것 같은데, Read-only 페이지로 관리되어야 하는 것이 아닌지?
> 2. 코드는 숫자로만, 타입을 ERROR_CASE_NAME 규칙으로 생성, 메세지는 2줄 이상 표시 가능, 가이드는 버튼 항목 + 처리 방식을 같이 표현할 수 있도록 구성

**변경사항:**
- 액션 컬럼(편집/삭제) 제거 → Read-only
- 툴바 "에러 코드 추가" 버튼 제거
- GitHub 관리 안내 배너 추가 (보라색, "GitHub에서 수정하기 →")
- 코드: `ERR_1001` → `1001` (숫자)
- 타입: `AUTH_TOKEN_EXPIRED` 등 ERROR_CASE_NAME 형식 적용
- 메시지: 2줄 (상황 설명 + 사용자 안내)
- 가이드: [버튼칩] + 처리 방식 설명

---

## 4. 피드백 2차

**User:**
> 1. 최근 수정일 우측 빈 컬럼값 삭제
> 2. 타입 배경 color값 삭제
> 3. 텍스트 크기 통일(noto sans, 12pt), 기본 노출 항목 15개로 고정
> 4. 가이드는 2개 나오는 Case도 고려해서 Chip | 문구 가로로 배치
> 5. 서비스 컬럼 추가(결제, 주문서, 인증, ...)

**변경사항:**
- 컬럼 총합 = 테이블 너비 1360px 정합
- 타입 배경색 제거
- 전체 텍스트 Noto Sans KR 12pt 통일
- 15개 데이터 행으로 확장 (인증/결제/공통/주문서/지갑)
- 가이드: `[칩] 설명문구` 가로 배치, 2개 가이드는 2줄 수직 스택
- 서비스 컬럼 추가

**최종 컬럼 (6개, 1360px):**
코드(80) / 타입(220) / 서비스(120) / 메시지(300) / 원인(190) / 가이드(450)

---

## 5. 피드백 3차

**User:**
> 1. Pagination에 n개씩 보기 선택을 Optional하게 할 수 있도록 select 항목 제공 (10, 15, 30, 50, 100, 150)
> 2. 필드 크기보다 Row 높이가 낮아서 깨져보이는 현상이 있는데, 필드 크기를 Row 높이가 hug해서 짤려보이는 현상 없게 수정
> 3. 가이드 버튼 Chip 높이 축소(세로로 너무 긴 형태)
> 4. 컬럼 너비 적절히 조절 필요
> 5. 최근 수정일 컬럼 삭제하고 다른 컬럼들 너비 더 여유롭게 조정

**변경사항:**
- 최근 수정일 컬럼 삭제, 가이드 450px으로 확장
- Row height: 1-guide 56px, 2-guide 80px (HUG 방식)
- 칩 paddingTop/Bottom 1px (컴팩트)
- 페이지네이션 `15개씩 ▾` 드롭다운 추가 (옵션: 10/15/30/50/100/150)

---

## 6. 피드백 4차

**User:**
> 1. 페이지네이션 항목 클릭하면 선택 가능한거라 기본 화면에서는 안보여도 될 것 같아.
> 2. 타입 컬럼에 색상값도 없어도 될 것 같아

**변경사항:**
- 페이지네이션 옵션 목록(펼쳐진 상태) 제거 → 드롭다운 버튼만 노출
- 타입 컬럼 텍스트 색상 제거 → 일반 텍스트 색상으로 통일
