## 주요 설계 결정

### Read-only 페이지
- 에러 코드는 GitHub에서 관리 → 즉각 수정/삭제 불가
- 관리 페이지는 조회 전용 구성
- 수정이 필요한 경우 PR을 통해 반영

### 데이터 규칙
- **code**: 숫자만 (1001, 2001 …)
- **type**: `ERROR_CASE_NAME` 형식
- **message**: 2줄 구성 (상황 설명 + 사용자 안내)
- **guide**: 버튼 1~2개, 각각 처리 방식 포함

### 페이지 구성 (컬럼)
| 컬럼 | 너비 |
|------|------|
| 코드 | 80px |
| 타입 | 220px |
| 서비스 | 120px |
| 메시지 | 300px |
| 원인 | 190px |
| 가이드 | 450px |

### 페이지네이션
- 기본 15개씩 보기
- 선택 옵션: 10 / 15 / 30 / 50 / 100 / 150
