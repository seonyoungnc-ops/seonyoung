## ❌ 개선 전 핵심 문제 (v1)

| # | 문제 | 심각도 |
|---|---|---|
| 1 | 결제수단 선택이 별도 팝업 3~4단계로 분리 | 🔴 Critical |
| 2 | NCPAY 등록 유도가 결제 플로우 진입 시 강제 노출 | 🔴 Critical |
| 3 | 결제 중 충전 금액/코인 정보 소실 (컨텍스트 단절) | 🟠 High |
| 4 | 이전 결제수단 기억/추천 없음 (매번 재탐색) | 🟠 High |
| 5 | 업셀링 로직 부재 (ARPU 기회 손실) | 🟡 Medium |
| 6 | 최종 확인 화면에 결제수단 미노출 | 🟡 Medium |

**실측 클릭 수: 최소 6~8회**

| 항목 | 개선 전 | 개선 후 |

| 하단 CTA 고정 | ❌ 없음 | ✅ 적용 |
| 전환율 예상 변화 | 기준 | +15~25% |

## ⚠️ 개선 후 남은 문제 (v2 → v3 과제)

| # | 문제 | 개선 방향 | 우선순위 |
|---|---|---|---|
| 1 | 중국어 약관 텍스트 혼재 | 국가별 서비스 분기 처리 | 🔴 즉시 |
| 2 | 금액 선택 영역 기본 접힘 | 진입 시 펼친 상태 기본값 | 🟠 단기 |
| 3 | 마지막 결제수단 미기억 | 자동 선택 + "지난번: OO카드" 표시 | 🟠 단기 |
| 4 | 결제수단 탭 위계 비일관 | 5개 탭 완전 통일 (NCPAY 포함) | 🟠 단기 |
| 5 | 카드사 드로어 화면 70% 점유 | 인라인 아코디언으로 전환 | 🟡 중기 |
| 6 | 신한카드 배너 맥락 불일치 | 신용카드 탭 선택 시에만 노출 | 🟡 중기 |
| 7 | 충전 한도 텍스트만 존재 | 프로그레스 바 + 경고 색상 | 🟡 중기 |
| 8 | 소득공제 정보 직관성 부족 | 체크 형태 간소화 + 모달 상세 | 🟢 장기 |
