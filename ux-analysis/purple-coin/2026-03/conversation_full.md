## 대화 2 — 실화면 기반 분석 (개선 전 화면)

### 화면 플로우 (7단계)
```
[화면1] 충전 메인 → 얼마나 충전할까요? (금액 선택)
[화면2] 금액 선택 완료 → 200,000원 / 202,000코인 확인
[화면3] 결제수단 선택① → NCPAY 등록 유도 (카드 미등록 상태)
[화면4] 결제수단 선택② → 일반결제 > 간편결제 탭 > 카드사 선택
[화면5] 결제수단 선택③ → 신용카드 탭 선택 → 카드사 그리드
[화면6] 결제수단 선택④ → 현대카드 선택 완료 → [선택] 버튼 활성
[화면7] 염성 확인 → 결제 정보 + 충전하기 버튼
```

**실측 클릭 수: 최소 6~8회**

### 주요 문제점

#### Nielsen Heuristic 위반
- **#1 Visibility of System Status**: 보유 코인 잔액이 즉시 보이지 않음
- **#4 Consistency & Standards**: 화면 간 디자인 언어 불일치
- **#6 Recognition over Recall**: 최근 구매 이력/추천 부재
- **#8 Aesthetic & Minimalist Design**: 결제수단 팝업 정보 과밀

#### 사용자 행동 방해 요소
- 화면3: NCPAY 등록 유도가 실제 결제 진행보다 먼저 노출
- 화면4→5: 탭 전환 필요 여부를 직관적으로 알기 어려움
- 화면6: [선택] 버튼 활성화까지 시각적 피드백 약함
- 화면7: 염성 확인 화면에서 결제수단 정보 미노출

#### 정보 구조(IA) 문제
```
현재 IA:
충전 팝업
├── 금액 선택 (화면1-2)
└── 결제수단 선택 [별도 팝업] ← 컨텍스트 단절
    ├── NCPAY 탭
    │   └── 등록하기 (미등록 시) ← 목적 이탈
    └── 일반결제 탭
        ├── 간편결제 (로고 그리드)
        ├── 신용카드 (카드사 그리드) ← 탭 뎁스
        └── 체크카드 (카드사 그리드)
```

### 개선 아이디어 (v1)
1. **원클릭 재충전 패턴** — 이전 결제 이력 기반 1클릭 완료
2. **스마트 결제수단 추천** — 최근 사용 순서 상위 3개 노출
3. **충전 금액 + 결제수단 통합 1-Screen** — 단일 팝업 구조
4. **NCPAY 등록 유도 타이밍 최적화** — 결제 완료 후 넛지
5. **보너스 코인 업셀링 모듈** — 금액 선택 영역 하단 배치
6. **충전 후 잔액 실시간 프리뷰** — 금액 선택 즉시 업데이트

## 대화 3 — 개선 화면 분석 (v2)

### 개선 화면 플로우 (3가지 뷰, 단일 화면)
```
[뷰A] 기타결제(가상계좌) 선택 상태
[뷰B] 간편결제(카카오페이) 선택 상태
[뷰C] 신용카드 선택 → 카드사 드로어 노출 상태
```

**클릭 수 변화: 6~8회 → 3~4회 (약 50% 단축)**

### 해결된 문제 ✅
| 항목 | 평가 |
|---|---|
| 금액+결제수단 단일 화면 통합 | ✅ 완전 해결 |
| 결제 중 보유코인/한도 상단 고정 | ✅ 완전 해결 |
| 결제수단 탭 구조 인라인 통합 | ✅ 구조 개선 |
| NCPAY 등록 강요 선노출 제거 | ✅ 완전 해결 |
| 하단 고정 CTA | ✅ 완전 해결 |

### 잔존/신규 문제점

#### 🔴 문제 1 — 결제수단 탭 위계 혼란
- NCPAY가 탭 형태가 아닌 별도 버튼으로 노출되다가 화면마다 다르게 표현
- Nielsen #4 Consistency 위반

#### 🔴 문제 2 — 금액 선택 영역의 접힘 구조
- "얼마를 충전할까요? ∨" → 접혀 있는 상태로 진입
- 핵심 행동(금액 선택)이 숨겨진 상태 → Nielsen #1 Visibility 위반

#### 🔴 문제 3 — 카드사 드로어의 정보 밀도
- 드로어가 화면 하단 70% 차지 → 결제 정보 영역 가림
- EVENT 뱃지 기준 불명확
- 우리카드 마이그레이션 안내가 결제 플로우 중간 노출

#### 🔴 문제 4 — 프로모션 배너 위치와 시각 충돌
- 가상계좌/카카오페이 선택 중 신한카드 배너 노출 (맥락 불일치)
- "바로가기" CTA가 충전하기 버튼보다 먼저 시선 유도

#### 🔴 문제 5 — 소득공제 계좌 정보 노출 위치
- 전화번호 형식의 소득공제 정보가 직관적이지 않음
- "변경" 링크 탭 가능 영역 부족

#### 🔴 문제 6 — 중국어 약관 텍스트 혼재
- 한국어 서비스에 중국어 안내 텍스트 노출
- 국가별 서비스 분기 로직이 UI에 노출

#### 🔴 문제 7 — 이번 달 충전한도 정보 활용 부재
- 텍스트만 존재, 프로그레스 바 없음
- 한도 초과 시 사전 경고 없음

### v3 개선 아이디어
1. **금액 선택 + 실시간 코인 프리뷰 강화** — 진입 시 펼친 상태
2. **결제수단 "마지막 사용" 자동 선택** — 재구매 클릭 2회로 단축
3. **이벤트 카드 맥락 분리** — 혜택 내용 직접 노출
4. **소득공제 정보 간소화** — 체크 형태 + 모달 상세
5. **자동 충전 배지 CTA 개선** — 충전 완료 후 인라인 넛지

## 최종 개선 로드맵

### v1 → v2 → v3 클릭 수 변화
| 버전 | 클릭 수 | 화면 수 | 전환율 예상 |
|---|---|---|---|
| v1 (개선 전) | 6~8회 | 7개 | 기준 |
| v2 (개선 후) | 3~4회 | 1개 | +15~25% |
| v3 (목표) | 2~3회 | 1개 | +30~40% |

### v3 우선순위
1. **즉시**: 중국어 텍스트 국가 분기 처리
2. **단기**: 금액 선택 기본 펼침 + 마지막 결제수단 자동 선택
3. **중기**: 카드사 드로어 → 인라인 아코디언 + 프로모션 맥락화
